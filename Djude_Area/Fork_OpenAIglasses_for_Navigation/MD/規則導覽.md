# 規則導覽索引

> **Agent 使用方式：** 不等使用者輸入指令。根據使用者的自然語言自動偵測情境，查下表找對應流程，只載入必要檔案。

---

## 自然語言 → 自動執行流程

| 使用者說的話（關鍵詞） | 自動執行流程 | 需讀的額外檔案 |
|----------------------|------------|--------------|
| **對話開始時（任何任務前）** | ① 檢查 `_交接資料/` 修改時間，有更新立即告知 ② git pull → /handover（處理 voice_missing_log/） | `feedback_session_start_checklist.md` |
| **push／推上去／送出** | /pre-commit → git add/commit/push → 確認 log | — |
| **刪掉／移除／刪除** | 執行 /delete skill 四步驟確認（誰引用？有備份？影響範圍？）→ 確認才刪 | — |
| **改／修／修正／修復** | 讀現況快照 → 查本表 → 修改 → 測試直到無誤 → 更新快照 | 依情境（見下方） |
| **啟動／跑起來** | `uv run python start_multi_device.py`（絕不用 app_main.py） | `feedback_startup_command.md` |
| **測試／跑一下／檢查** | 依準則 6 選驗證方式，全部通過才回報完成 | — |
| **架構／Port／模組** | 讀 /arch skill 內容回答 | — |
| **對話結束／沒事了** | 更新現況快照 → 更新 MEMORY.md → 告知下一步 | `MD/現況快照.md` |

## 行為準則速查（詳細在 `CLAUDE.md`）

| # | 準則 | 一句話核心 |
|---|------|-----------|
| 1 | 動手前先思考 | 假設說出來，不確定就先問，多種解法全列出 |
| 2 | 簡潔優先 | 最少程式碼，不加多餘功能或彈性設計 |
| 3 | 精準修改 | 只動必要處，不順便重構，不刪別人的殭屍碼 |
| 4 | 深度理解、持續更新交接 | 先讀現況快照，完成後更新 MD 和 MEMORY |
| 5 | 目標導向執行 | 多步驟任務先列計畫，每步有驗證方式 |
| 6 | 測試直到無誤才結束 | 每次更新後自行測試，有錯立即修正再測，全過才算完成 |
