## 第一章：系統功能介紹

### 即時物件辨識功能

本系統採用 **YOLOE-26** 開放詞彙實例分割模型，能夠即時辨識畫面中的物件。系統會在辨識到的物件周圍繪製邊界框，並顯示物件名稱與信心度。

**核心特色：**
- **開放詞彙偵測**：可偵測任何物品，不限於預訓練類別
- **文字提示功能**：使用 `set_classes()` 指定要偵測的物品名稱
- **實例分割**：同時輸出邊界框和分割遮罩
- 即時處理影像，每秒可達 15-25 幀
- 支援 GPU 加速 (NVIDIA CUDA)
- 自動將英文標籤轉換為中文

### YOLOE-26 開放詞彙功能

YOLOE-26 是 Ultralytics 的開放詞彙實例分割模型，具有以下特點：

| 特性 | 說明 |
|------|------|
| 開放詞彙 | 可偵測任何物品，不限於 COCO 80 類 |
| 文字提示 | 透過 `DETECTION_CLASSES` 環境變數設定要偵測的物品 |
| 實例分割 | 同時輸出邊界框和分割遮罩 |
| 模型大小 | YOLOE-26S 適合 RTX 3060 |

**使用範例：**
```env
# 在 .env 檔案中設定要偵測的物品（用逗號分隔）
DETECTION_CLASSES=cell phone,bottle,cup,handbag,backpack,remote,mouse,keyboard,laptop,wallet,keys
```

### ESP32 串流功能

透過 XIAO ESP32-S3 Sense 開發板搭配攝影機，系統可以無線傳輸影像到筆電進行辨識。

**技術規格：**
- 使用 HTTP MJPEG 協議
- 低延遲設計，延遲約 150-300ms
- 支援 WiFi 連線
- **支援 mDNS 自動發現** - 無需手動設定 IP

### 資料庫記錄功能

系統會自動將辨識結果存入 SQLite 資料庫，方便日後查詢與統計。

**記錄內容：**
- 物品名稱 (中英文)
- 辨識信心度
- 首次與最後出現時間
- 累積辨識次數

### 自定義訓練功能

預訓練模型無法辨識的物品 (如台灣紙鈔、鑰匙)，可以透過自定義訓練來新增。

**訓練流程：**
1. 收集訓練圖片 (建議 200 張以上)
2. 使用標註工具標記物件
3. 訓練自定義模型
4. 載入新模型進行辨識

### 支援的物品類別列表

**YOLOE-26 開放詞彙功能：**

YOLOE-26 是開放詞彙模型，可以偵測任何物品，不限於預訓練類別。只需在 `.env` 檔案中設定 `DETECTION_CLASSES` 即可指定要偵測的物品。

**常用偵測類別（可直接使用）：**

| 物品 | 英文名稱 | 備註 |
|------|----------|------|
| 手機 | cell phone | 直接可用 |
| 水瓶 | bottle | 直接可用 |
| 杯子 | cup | 可用於可樂罐 |
| 手提包/錢包 | handbag | 直接可用 |
| 背包 | backpack | 直接可用 |
| 遙控器 | remote | 直接可用 |
| 滑鼠 | mouse | 直接可用 |
| 鍵盤 | keyboard | 直接可用 |
| 筆電 | laptop | 直接可用 |
| 錢包 | wallet | 直接可用 |
| 鑰匙 | keys | 直接可用 |

**開放詞彙優勢：**

| 傳統 YOLO 模型 | YOLOE-26 開放詞彙模型 |
|----------------|----------------------|
| 只能偵測 80 種 COCO 類別 | 可偵測任何物品 |
| 需要自定義訓練才能偵測新類別 | 直接輸入物品名稱即可 |
| 訓練需要大量標註資料 | 無需訓練，即開即用 |

> **提示：** YOLOE-26 的開放詞彙功能讓您無需進行自定義訓練即可偵測鑰匙、錢包等物品。只需將物品名稱加入 `DETECTION_CLASSES` 即可。

## 第三章：安裝步驟

### Step 1: 安裝 uv

uv 是快速的 Python 套件管理器。

```powershell
# Windows PowerShell
pip install uv
```

### Step 2: 建立虛擬環境

```powershell
# 進入專案目錄
cd D:\GitHub_Project\yollo_E

# 建立虛擬環境
uv venv

# 啟動虛擬環境
.\.venv\Scripts\activate
```

### Step 3: 安裝套件

```powershell
# 安裝所有專案套件
uv sync
```

### Step 4: 設定 .env

在專案根目錄建立或編輯 `.env` 檔案：

```env
# WiFi 設定
WIFI_SSID=你的WiFi名稱
WIFI_PASSWORD=你的WiFi密碼

# ESP32 設定 (使用 mDNS 自動發現，無需手動查 IP)
ESP32_HOSTNAME=yollo
# ESP32_IP=               # 可選，若 mDNS 失敗時可手動填入 IP 作為備援

# 模型設定 - YOLOE-26 開放詞彙實例分割模型
MODEL_PATH=yoloe-26s-seg.pt

# YOLOE 開放詞彙偵測類別（可自訂任何物品，留空則使用模型內建類別）
DETECTION_CLASSES=cell phone,bottle,cup,handbag,backpack,remote,mouse,keyboard,laptop,wallet,keys
```

## 第四章：ESP32 燒錄教學

### Arduino IDE 設定

1. 開啟 Arduino IDE
2. 點選「檔案」→「偏好設定」
3. 在「額外的開發板管理員網址」填入：
   ```
   https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
   ```
4. 點選「工具」→「開發板」→「開發板管理員」
5. 搜尋 "esp32"，安裝版本 **3.3.6** 或以上

### 開發板選項設定

開啟 `esp32/yollo_E.ino`，設定以下選項：

| 設定項目 | 值 |
|----------|---|
| Board | XIAO_ESP32S3 |
| Flash Mode | QIO 80MHz |
| Flash Size | 8MB (64Mb) |
| **PSRAM** | **OPI PSRAM** (必須開啟!) |
| Upload Mode | UART0 / Hardware CDC |

### 上傳步驟

1. 使用 USB 線連接 XIAO ESP32-S3 Sense 到電腦
2. 在 Arduino IDE 選擇正確的 COM 埠
3. 點選上傳按鈕 (→)
4. 等待編譯與上傳完成

### 取得 IP 方法

ESP32 支援 **mDNS 自動發現**功能，啟動後會自動廣播主機名 `yollo.local`，無需手動查詢 IP。

#### 方法一：使用 mDNS 自動發現 (推薦)

1. 上傳完成後，點選「工具」→「序列監視器」
2. 設定鮑率為 **115200**
3. 按下 ESP32 上的 RST 重置按鈕
4. 序列監視器會顯示連線資訊，例如：
   ```
   WiFi connected!
   mDNS responder started
   Hostname: yollo.local
   Stream URL (mDNS): http://yollo.local/stream
   ```
5. **無需任何設定**，系統會自動使用 `yollo.local` 連接

#### 方法二：手動查詢 IP (備援)

若 mDNS 無法正常運作，可手動查詢 IP：

1. 開啟序列監視器查看輸出的 IP 位址
2. 將取得的 IP (如 192.168.1.105) 填入 `.env` 檔案：
   ```env
   ESP32_IP=192.168.1.105
   ```

> **注意：** Windows 10/11 原生支援 mDNS，大多數情況下無需手動設定 IP。

## 第五章：執行辨識系統

### 基本執行指令

```powershell
# 啟動虛擬環境
.\.venv\Scripts\activate

# 使用 ESP32 串流執行
uv run python src/main.py --source esp32
```

### 使用 Webcam 測試

如果沒有 ESP32，可以使用筆電內建的 Webcam 測試：

```powershell
uv run python src/main.py --source webcam
```

### 進階參數說明

| 參數 | 說明 | 預設值 |
|------|------|--------|
| `--source` | 影像來源 (esp32/webcam) | esp32 |
| `--model` | 模型路徑 | yoloe-26s-seg.pt |
| `--custom` | 使用自定義模型 | False |
| `--confidence` | 信心度門檻 (0.0-1.0) | 0.5 |
| `--no-db` | 不儲存到資料庫 | False |
| `--save-images` | 儲存辨識圖片 | False |
| `--no-display` | 不顯示視窗 | False |

**使用範例：**

```powershell
# 調高信心度門檻 (減少誤判)
uv run python src/main.py --source esp32 --confidence 0.7

# 儲存辨識圖片
uv run python src/main.py --source esp32 --save-images

# 使用自定義模型
uv run python src/main.py --source esp32 --custom
```

## 第七章：自定義訓練

> **注意：** YOLOE-26 具有開放詞彙功能，大多數情況下可以直接透過 `DETECTION_CLASSES` 設定要偵測的物品，無需進行自定義訓練。只有在開放詞彙效果不佳時才需要自定義訓練。

### 收集資料

使用系統提供的資料收集工具：

```powershell
# 收集鑰匙圖片 (建議 200 張以上)
uv run python -c "
from src.training.data_collector import DataCollector
collector = DataCollector(class_name='keys')
collector.collect_from_webcam(target_count=200)
"

# 收集台灣紙鈔
uv run python -c "
from src.training.data_collector import DataCollector
collector = DataCollector(class_name='100ntd')
collector.collect_from_webcam(target_count=200)
"
```

### 標註資料

收集完圖片後，需要進行標註：

**推薦工具：**
- [Roboflow](https://roboflow.com) - 線上標註，支援匯出 YOLO 格式
- [LabelImg](https://github.com/tzutalin/labelImg) - 本地標註工具

**標註步驟：**
1. 上傳收集的圖片
2. 為每張圖片中的目標物件繪製邊界框
3. 指定類別標籤
4. 匯出為 YOLO 格式

### 訓練模型

```powershell
# 使用準備好的資料集進行訓練
uv run python src/training/train_custom.py --data dataset.yaml --epochs 100
```

訓練完成後，模型會儲存在 `models/custom/` 目錄。

### 使用自定義模型

```powershell
uv run python src/main.py --source esp32 --custom
```

## 第八章：常見問題 FAQ

### Q1: ESP32 無法連接 WiFi

**可能原因與解決方案：**

1. **WiFi 帳密錯誤**
   - 確認 `.env` 中的 SSID 和密碼正確
   - 注意大小寫區分

2. **不同網段**
   - 確認 ESP32 和筆電連接同一個 WiFi

3. **訊號不良**
   - 縮短 ESP32 與路由器的距離
   - 移除干擾源

4. **重新燒錄**
   - 嘗試重新燒錄 ESP32 程式

### Q2: 串流延遲過高

**解決方案：**

1. 降低 ESP32 的影像解析度 (修改 `config.h`)
2. 降低 JPEG 壓縮品質
3. 確保 WiFi 訊號良好
4. 縮短 ESP32 與路由器距離

### Q3: YOLO 辨識速度慢

**解決方案：**

1. **確認 GPU 運作**
   - 程式啟動時應顯示 CUDA 相關資訊

2. **使用較小模型**
   - YOLOE-26S 是目前最小的 YOLOE 模型，適合 RTX 3060

3. **降低影像解析度**
   - 修改 ESP32 的串流解析度

4. **更新驅動程式**
   - 確認 NVIDIA 驅動程式為最新版本

### Q4: 無法辨識特定物品（如台灣紙鈔）

**說明：** YOLOE-26 是開放詞彙模型，可以嘗試直接將物品名稱加入 `DETECTION_CLASSES`。

**解決方案：**

1. **先嘗試開放詞彙功能**
   - 將物品名稱加入 `DETECTION_CLASSES`
   - 例如：`DETECTION_CLASSES=cell phone,bottle,taiwan dollar,100 ntd bill`

2. **如果開放詞彙效果不佳，再進行自定義訓練**
   - 收集 200 張以上圖片
   - 使用 Roboflow 進行標註
   - 訓練自定義模型
   - 使用 `--custom` 參數執行

### Q5: 程式無法啟動

**檢查清單：**

1. 虛擬環境是否已啟動？
   ```powershell
   .\.venv\Scripts\activate
   ```

2. 套件是否已安裝？
   ```powershell
   uv sync
   ```

3. `.env` 檔案是否存在且設定正確？

4. 查看錯誤訊息，依提示修正

### Q6: 辨識準確率低

**解決方案：**

1. **調整信心度門檻**
   - 降低門檻可偵測更多物件
   - 提高門檻可減少誤判

2. **改善光線**
   - 確保拍攝環境光線充足

3. **調整攝影機角度**
   - 避免逆光或強烈陰影

4. **使用自定義訓練**
   - 针對特定物品進行訓練
