## 工作規則（必須遵守）

### 語言規範
- 所有回覆一律使用 **繁體中文**
- 所有程式碼註釋必須使用 **繁體中文**

### 絕對要遵守
- 絕對不能洩漏任何與我相關個資
- 絕對不能洩漏任何與我相關訊息

### 更新後的操作規範
- **每次程式碼更新完成後，必須提供重啟指令**，讓使用者手動重新啟動服務
- **每次更新完成後，必須使用 `git commit` 提交變更**
- **`git commit` 訊息必須使用英文撰寫**，以維持專業性與國際可讀性
- 禁止自動重啟伺服器，由使用者手動執行

### 工作日誌規範
- **每次完成任務後，必須在工作日誌 (`WORK_LOG.md`) 中記錄並總結內容**
- 記錄格式：日期、任務摘要、修改的檔案、解決的問題、注意事項
- 目的：讓下一個 AI 助手或使用者能快速了解專案狀態，類似 `/compact` 的作用

## 環境隔離與安全規範

### 套件與系統環境
- 任何會影響 Python 套件或環境變數的操作（例如 `pip install`、`uv add` 等）
- 必須在 **Python 虛擬環境 (`.venv`) 或 Docker 容器** 中執行
- 禁止直接污染全域 Python 環境

### Python 開發規範
- Python 套件管理統一使用 `uv`
- 建立虛擬環境：`uv venv`
- 安裝套件：`uv add`
- 執行程式：`uv run`

### 使用 `.env` 作為全域敏感資訊配置

規範：

- 所有專案的敏感變數必須存放於 `.env`
- 使用 `python-dotenv` 或相同方式讀取環境變數

典型使用方式：

```python

import os

load_dotenv()

API_KEY = os.getenv("API_KEY")
PASSWORD = os.getenv("PASSWORD")
