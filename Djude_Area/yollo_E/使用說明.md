## 安裝步驟

### Step 1: 安裝 uv

```powershell
# Windows (PowerShell)
pip install uv
```

### Step 2: 建立虛擬環境

```powershell
# 進入專案目錄
cd d:\GitHub_Project\yollo_E

# 建立虛擬環境
uv venv

# 啟動虛擬環境
.\.venv\Scripts\activate

# 安裝所有套件
uv sync
```

### Step 3: 設定環境變數

編輯 `.env` 檔案（已預先建立）：

```env
# WiFi 設定
WIFI_SSID=你的WiFi名稱
WIFI_PASSWORD=你的WiFi密碼

# ESP32 設定 (燒錄後填入實際 IP)
ESP32_IP=192.168.1.100

# 模型設定
MODEL_PATH=yoloe-26s-seg.pt
```

## ESP32 燒錄

### Step 1: 安裝 ESP32 開發板

1. 開啟 Arduino IDE
2. 檔案 → 偏好設定
3. 額外的開發板管理員網址：
   ```
   https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
4. 工具 → 開發板 → 開發板管理員
5. 搜尋 "esp32"，安裝版本 3.3.6 或以上

### Step 2: 設定開發板

開啟 `esp32/yollo_E.ino`，設定以下選項：

| 設定項目 | 值 |
|----------|---|
| Board | XIAO_ESP32S3 |
| Flash Mode | QIO 80MHz |
| Flash Size | 8MB (64Mb) |
| **PSRAM** | **OPI PSRAM** (必須開啟!) |
| Upload Mode | UART0 / Hardware CDC |

### Step 3: 上傳程式

1. USB 連接 XIAO ESP32-S3 Sense
2. 選擇正確的 COM 埠
3. 點擊「上傳」
4. 等待上傳完成

### Step 4: 取得 IP 位址

1. 開啟序列監視器 (工具 → 序列監視器)
2. 設定鮑率為 **115200**
3. 按下 ESP32 的 RST 重置按鈕
4. 記下顯示的 IP 位址，例如：
   ```
   Camera Ready! Use 'http://192.168.1.105/stream' to stream video
   ```

### Step 5: 更新 .env

將取得的 IP 填入 `.env`：
```env
ESP32_IP=192.168.1.105
```

## 執行辨識系統

### 基本執行

```powershell
# 啟動虛擬環境
.\.venv\Scripts\activate

# 使用 ESP32 串流
uv run python src/main.py --source esp32
```

### 使用 Webcam 測試

```powershell
# 使用本地 Webcam (不需 ESP32)
uv run python src/main.py --source webcam
```

### 進階選項

```powershell
# 調整信心度門檻
uv run python src/main.py --source esp32 --confidence 0.7

# 不顯示視窗 (背景執行)
uv run python src/main.py --source esp32 --no-display

# 儲存辨識圖片
uv run python src/main.py --source esp32 --save-images

# 使用自定義模型
uv run python src/main.py --source esp32 --custom
```

## 常用指令

### 按鍵操作

| 按鍵 | 功能 |
|------|------|
| `q` | 結束程式 |
| `s` | 儲存當前畫面 |

### 命令列參數

| 參數 | 說明 | 預設值 |
|------|------|--------|
| `--source` | 影像來源 (esp32/webcam) | esp32 |
| `--model` | 模型路徑 | yoloe-26s-seg.pt |
| `--custom` | 使用自定義模型 | False |
| `--confidence` | 信心度門檻 (0.0-1.0) | 0.5 |
| `--no-db` | 不儲存到資料庫 | False |
| `--save-images` | 儲存辨識圖片 | False |
| `--no-display` | 不顯示視窗 | False |

## 常見問題

### Q1: ESP32 無法連接 WiFi

**解決方案**:
1. 確認 WiFi SSID 和密碼正確
2. 確認 ESP32 和筆電在同一網段
3. 檢查 WiFi 訊號強度
4. 重新燒錄 ESP32 程式

### Q2: 串流延遲過高

**解決方案**:
1. 降低 ESP32 的影像解析度 (修改 config.h)
2. 降低 JPEG 品質
3. 確保 WiFi 訊號良好
4. 縮短 ESP32 與路由器距離

### Q3: YOLO 辨識速度慢

**解決方案**:
1. 確認使用 GPU (CUDA)
2. 使用 YOLOE-26S 模型（已優化）
3. 降低影像解析度
4. 檢查 GPU 驅動程式

### Q4: 無法辨識台灣紙鈔

**解決方案**:
台灣紙鈔不在 COCO 資料集中，需要自定義訓練：
1. 收集 200+ 張紙鈔圖片
2. 使用 Roboflow 標註
3. 訓練自定義模型
4. 使用 `--custom` 參數執行

### Q5: 程式無法啟動

**解決方案**:
1. 確認虛擬環境已啟動 (`.\.venv\Scripts\activate`)
2. 確認套件已安裝 (`uv sync`)
3. 檢查 .env 檔案設定
4. 查看 error log
