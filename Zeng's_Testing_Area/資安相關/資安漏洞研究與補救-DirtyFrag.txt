Dirty Frag 的最新 Linux 系統雙重漏洞（CVE-2026-43284 和 CVE-2026-43500）。

以下是影片的核心重點總結：

漏洞原理：這個漏洞與之前的 Copy Fail 類似，但在處理網路通訊（解密數據包）時，系統的優化機制錯誤地將解密結果直接寫回了唯讀檔案的「頁面快取 (Page Cache)」中，這導致攻擊者可以直接竄改記憶體中的檔案臨時副本 [01:14]。

雙重漏洞互補通殺：

IPsec 變體：存在長達 9 年。透過 IPsec 加密通訊協定觸發，可竄改 su 指令的記憶體副本以取得 root 權限 [02:42]。但觸發它需要建立使用者命名空間 (user namespace) 的權限，在有嚴格安全策略的系統（如 Ubuntu）上無法單獨觸發 [01:44]。

RPC 變體：存在近 3 年。透過 RPC 協定觸發，不需要特殊權限，會直接清空 /etc/passwd 檔案中 root 的密碼欄位，實現無密碼登入 [02:45]。多數 Linux 發行版預設未安裝此核心模組，但 Ubuntu 預設有安裝 [02:03]。

這兩個漏洞單獨利用都有盲區，但結合起來就能涵蓋幾乎所有主流的 Linux 發行版。

伺服器與容器化威脅：這對於日常維護 Ubuntu Server、Debian 或架設 Docker 容器的環境來說，是一個需要特別防範的安全死角。攻擊者能輕易透過修改記憶體快取達成容器逃逸 (Container Escape)，直接從單一容器打穿並獲取底層宿主機 (Host) 的 root 權限 [03:20]。

修復狀態與緩解措施：

先前的 Copy Fail 修補程式對 Dirty Frag 完全無效，因為兩者走的是完全不同的核心模組 [02:32]。

目前 IPsec 變體的修補已合併至主線核心，但 RPC 變體至今仍未有修補程式，且漏洞的利用程式碼 (PoC) 已經在網路上公開流傳 [03:30]。

臨時緩解方案：可以停用並卸載與 IPsec 和 RPC 相關的三個核心模組，但這會導致 VPN 或部分網路檔案系統無法正常運作，在對外服務套用前需要謹慎評估 [03:43]。
