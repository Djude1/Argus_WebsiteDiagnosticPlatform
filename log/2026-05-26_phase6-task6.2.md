# Phase 6 Task 6.2：公告系統 API 端點

**日期**：2026-05-26
**操作者**：Claude（Session 3）

## 變更內容

- `backend/apps/admin_api/serializers.py`：
  - import `Announcement` model
  - 新增 `AnnouncementSerializer`（ModelSerializer，fields 含 id/title/content/type/active_days/is_active/created_at，`created_at` 唯讀）
- `backend/apps/admin_api/views.py`：
  - import `Announcement` + `AnnouncementSerializer`
  - 新增 `active_announcements()`：GET，IsAuthenticated，回傳目前有效公告
  - 新增 `announcements_admin()`：GET（list 全部含停用）/ POST（建立），IsAdminUser
  - 新增 `announcement_detail()`：GET / PATCH / DELETE，IsAdminUser
- `backend/apps/admin_api/urls.py`：
  - 新增 3 條路由（`announcements/active/` / `announcements/` / `announcements/<pk>/`）

## 原因

26 項修正計畫 Phase 6 Task 6.2 — Dashboard 公告 Modal（Task 6.3）與後台公告管理 UI（Task 6.4）的後端先決條件。

## 影響範圍

- `apps.admin_api`：3 個新 endpoint（不影響既有 17 個 endpoint）
- 端點清單：
  - `GET /api/admin/announcements/active/`（任何登入使用者）→ `{"announcements": [...]}`
  - `GET /api/admin/announcements/`（staff）→ 全部
  - `POST /api/admin/announcements/`（staff）→ 建立
  - `GET /api/admin/announcements/<pk>/`（staff）
  - `PATCH /api/admin/announcements/<pk>/`（staff）
  - `DELETE /api/admin/announcements/<pk>/`（staff）

## 驗證方式

| 項目 | 結果 |
|---|---|
| `manage.py check` | ✅ no issues |
| `manage.py test apps.admin_api -v 1` | ✅ 既有 34 tests pass |
| RequestFactory 行為驗證（10 case） | 全 ✅ |

行為驗證涵蓋：
1. 普通登入者 GET active → 200，看到法律公告
2. 未登入 GET active → 401
3. staff GET admin list → 200
4. 非 staff GET admin list → 403
5. staff POST 建臨時公告 → 201
6. staff GET detail → 200
7. staff PATCH 停用 → 200，`is_active=False`
8. 停用後普通使用者 GET active → 該公告消失（business rule 對齊 `is_currently_active()`）
9. staff DELETE → 204
10. GET 不存在 pk → 404

## 計畫偏離說明

- 計畫 Step 2 函式簽名用 `[IsAdminUser]`（直接 import），既有檔案慣用 `[permissions.IsAdminUser]`（DRF 內建）；遵循既有風格。
- 計畫 Step 2 `announcement_detail` 用 try/except 抓 DoesNotExist 回 404，改用 `get_object_or_404`（既有 view 慣用）。
