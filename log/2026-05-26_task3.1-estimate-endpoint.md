# Task 3.1：新增 POST /api/estimate/ 預掃描費用估算端點

**日期**：2026-05-26  
**操作者**：Claude

## 變更內容
- `backend/apps/scans/views.py`：在檔案末尾新增 `estimate_scan` view 函式，以及三個輔助函式（`_is_safe_url`、`_count_links`、`_try_sitemap`）
- `backend/apps/scans/urls.py`：匯入 `estimate_scan` 並在 urlpatterns 加入 `path("estimate/", estimate_scan, name="estimate-scan")`
- `backend/apps/scans/tests.py`：新增 `EstimateScanTests` 類別（3 項測試）

## 原因
Task 3.1 需求：在掃描建立前提供費用預估功能，讓使用者了解大約需要花費多少點數，而不實際扣點或寫入資料庫。

## 影響範圍
- 新增端點 `POST /api/estimate/`（掛在 `path("api/", include("apps.scans.urls"))` 下，實際路徑為 `/api/estimate/`）
- 完全不觸碰資料庫（不建立 ScanJob、不呼叫 billing）
- `_is_safe_url` 新增「主機名稱必須含點」驗證，拒絕 `not-a-url`、裸主機名稱等無效輸入

## 驗證方式
- `uv run python backend/manage.py test apps.scans.tests.EstimateScanTests -v 2`：3/3 pass
- `uv run python backend/manage.py test apps.scans -v 2`：全套迴歸測試通過（無 FAIL）
