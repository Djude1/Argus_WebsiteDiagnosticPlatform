# Task 3.3：發票 UI 重構——個人/公司分開設定，加統一編號欄位

**日期**：2026-05-26
**操作者**：Claude

## 變更內容

- `frontend/src/App.jsx`（~2947 行起）：將結帳 wizard Step 2 的發票設定區塊重構
  - 移除舊的 `wizard-field` 巢狀結構（`發票類型` + `電子發票載具` 兩個獨立欄位）
  - 改為單一 `billing-invoice-section` 容器，清楚呈現兩層選擇邏輯
  - Level 1：個人電子發票 vs 公司統一發票（互斥 radio）
  - Level 2（個人）：雲端 / 手機條碼 / 自然人憑證，非雲端時顯示 carrier_id 輸入框
  - Level 2（公司）：公司名稱輸入框 + 統一編號（8 碼，自動過濾非數字）
  - 切換 `invoice_type` 時同步重置 `carrier_type: "cloud"`、`carrier_id: ""`，避免殘留舊值
- `frontend/src/styles.css`（`billing-tx-table` 區塊之後）：新增對應 CSS 類別
  - `.billing-invoice-section`、`.billing-section-title`
  - `.billing-radio-group`、`.billing-radio-label`（含 `has-[:checked]` 高亮）
  - `.billing-carrier-section`、`.billing-company-section`
  - `.billing-error`

## 原因

Task 3.3 要求：以雙層設計取代舊版（舊版個人/公司與載具選項分屬不同欄位，視覺上容易混淆）。

## 影響範圍

- 僅影響 `/purchase` 結帳 wizard 的 Step 2（填寫發票資訊）
- Step 1（選方案）、Step 3（付款）完全不變
- `buyer` state 初始值、`validateStep2`、送出 payload 邏輯均未更動（已含 `company_name`、`tax_id`、`carrier_type`、`carrier_id`）
