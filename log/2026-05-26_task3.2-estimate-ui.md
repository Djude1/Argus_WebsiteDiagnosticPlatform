# Task 3.2：掃描表單加預估費用按鈕，移除最大頁數上限選項

**日期**：2026-05-26  
**操作者**：Claude

## 變更內容

### `frontend/src/App.jsx`
- 移除 `maxPages` / `setMaxPages` state（及 `SITE_PAGE_PRESETS` 常數）
- 移除「最大頁數上限」preset 按鈕組與自定義 number input 的 UI 區塊
- 整站掃描的 `max_pages` API 參數改為固定值 `500`
- `effectivePages` 計算改為 `scope === "single" ? 1 : 500`
- 整站卡片的 `scope-meta` 文字改為「最多 500 頁，依實際爬到頁數計費」
- 新增 `estimating` / `estimate` state 及 `handleEstimate()` async function
- URL input `onChange` 同時呼叫 `setEstimate(null)` 以清除舊估算結果
- URL input 下方加入「預估費用」按鈕與結果展示區（`scope !== "single"` 時才顯示）
- draft localStorage 同步移除 `maxPages` 欄位
- 送出成功後的 reset 移除 `setMaxPages(50)`，改為 `setEstimate(null)`

### `frontend/src/styles.css`
- 末尾新增 `.scan-estimate-row`、`.scan-estimate-btn`、`.scan-estimate-result`、`.scan-estimate-conf` 樣式
- 信心等級色條：high=綠、medium=黃、low=紅

## 原因

Task 3.2 需求：讓使用者在建立掃描前可預估費用（呼叫後端 `/api/estimate/`），
同時移除讓使用者手動設定最大頁數的輸入，改為系統固定 500 頁上限，降低 UX 複雜度。

## 影響範圍

- `ScanJobForm` 元件外觀與邏輯
- 整站掃描固定以 `max_pages=500` 送出 API（後端仍依實際爬到的頁數計費）
- 「預估費用」按鈕呼叫 `/api/estimate/`（Task 3.1 建立的端點）
