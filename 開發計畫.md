# Argus 開發計畫

## 專案理解摘要

**Argus** 是一個 SaaS 級網站健檢平台，整合：
- Playwright 全站爬蟲（同網域、深度 3 層）
- 四維靜態掃描（SEO / AEO / GEO / 被動資安）
- Hermes-Agent 動態 UX 測試
- 可互動視覺化工作區（截圖 + Canvas 高光）
- 結構化問題描述 Prompt（不產生修復程式碼）

**現狀**：MVP 後端與前端骨幹已完成並通過測試（27 項後端測試、前端 production build）；剩餘為 Docker 部署、前端路由、Phase 2 與部分測試。

## 外部參考技術採納清單

正式矩陣：`skills/argus-project/references/technology-adoption.md`

### 已納入 MVP
- **Scrapling 概念**：bounded concurrency、per-origin rate limit、timeout/retry、blocked detection、pause/resume checkpoint、cache replay；用 Playwright + Celery 實作，不引入 Scrapling runtime。
- **GEOFlow 概念**：任務佇列、失敗記錄、provider fallback、後台任務狀態與健康指標；不採納 PHP/Laravel 技術棧。
- **aeo-site / Optimeyes 概念**：0-100 分數、分類 breakdown、Top Actions、score gauge、優先處理排序、專業報告呈現。
- **awesome-geo 概念**：FAST framework、`llms.txt`、AI crawler robots 診斷、Schema/semantic HTML/SSR 可讀性、chunkability、entity clarity、evidence quality。
- **RTK**：大型命令輸出壓縮與 agent 工作規則整合。

### Phase 2/3 再做
- RAG/知識庫切片、embedding、素材庫、prompt library。
- keyword tracking、competitor analysis、brand mention/citation monitoring。
- 多語言分析與 API integration。

### 明確不採納
- 反爬繞過、指紋偽裝、代理池規避、未授權掃描。
- 外部專案的 PHP/Laravel、Express/Drizzle 等替代技術棧。
- 自動產生修復後程式碼或批量內容污染。

## 關鍵決策與風險

### 決策點
1. **Django vs FastAPI**：已選定 Django（需求文件固定）
2. **React vs Vue**：已選定 React（需求文件固定）
3. **AI API Provider**：MiniMax-M2.7 優先，GLM `glm-4.7-flash` / `glm-4.5-flash` 第二順位，Gemini 作分析備援；保留 Hermes-style Agent 行為架構與 tool calling 技術路線
4. **資料庫**：SQLite（dev）/ PostgreSQL（prod）
5. **任務佇列**：Celery + Redis（已固定）

### 風險點
1. **法律風險**：未經授權的爬蟲可能違法 → 必須嚴格實作授權同意書記錄
2. **性能風險**：Playwright 爬蟲可能耗時 → Celery 任務佇列 + 非同步
3. **成本風險**：Hermes Agent token 消耗 → 需要步數上限（20 步）
4. **座標映射複雜度**：DPR、fixed elements、viewport vs full-page → 需要小心測試
5. **大型截圖**：全頁截圖可能很大 → 需要壓縮或限制尺寸

## 法律與倫理硬性限制（CRITICAL）

### 授權確認機制
- 使用者送出網址前，必須勾選「我擁有此網站或已獲得書面授權測試」之同意書
- 後端記錄 IP、timestamp、user_id

### 爬蟲範圍
- 預設僅同網域(same-origin)
- 最大深度 3 層
- 最大頁數 50（可在後台調整）
- 遵守 `robots.txt`（預設開啟，可由 admin 關閉）

### 資安測試模式
- 預設為「被動偵測(Passive)」— 只分析 response header、HTML、cookie flags、CSP、表單缺少 CSRF token 等不發送惡意 payload 的檢測

### 主動測試（Active）限制
- 使用者額外勾選「我同意進行侵入式測試」
- 限制 RPS（每秒請求數）≤ 2
- 僅使用無破壞性 payload
- 所有請求加上自訂 User-Agent：`SiteSense-AI-Scanner/1.0 (authorized-audit)`

### AI 拒絕
- 若使用者輸入明顯第三方網站（google.com、銀行網域等），UI 顯示警告並要求重新確認
