# Security Policy

## Supported Versions

| Version | Supported | Maintenance |
|---------|-----------|-------------|
| 1.0.x   | ✅ Yes     | Active     |
| 0.9.x   | ⚠️ Limited | Critical only |
| < 0.9.0 | ❌ No      | Unsupported |

Only the latest stable version (1.x.x) receives full security updates and new features. Previous versions may receive critical security patches on a limited basis.

## Reporting a Vulnerability

We take security very seriously and appreciate your efforts to responsibly disclose vulnerabilities. This AI device is designed to assist visually impaired users, so security and privacy are paramount.

### How to Report

To report a security vulnerability, please send an email to **security@openaidevice.com** with the following subject line:

```
[SECURITY] [CVE ID - if known] Vulnerability Report - [Brief Description]
```

### What to Include in a Report

When reporting a vulnerability, please include as much of the following information as possible:

- **Description**: Detailed explanation of the vulnerability
- **Steps to Reproduce**: Clear steps to reproduce the issue
- **Expected vs Actual Behavior**: What should happen vs what actually happens
- **Environment**: Device model, OS version, AI model version, and any other relevant details
- **Impact Assessment**: Potential impact on user safety, privacy, or data
- **Proof of Concept (optional)**: Code snippets or detailed reproduction steps
- **Suggested Mitigation**: If you have ideas for fixing the issue

### Response Timeline

We follow this timeline for vulnerability reports:

1. **Acknowledgment**: Within 24 hours of receiving your report
2. **Initial Assessment**: Within 3 business days
3. **Resolution Update**: Weekly progress updates while investigating
4. **Fix Deployment**: Within 30 days for critical issues (may vary based on complexity)
5. **Credit**: You will be credited for your contribution (if desired)

### Disclosure Policy

#### Disclosure Timeline

- **Critical vulnerabilities**: Disclosed within 90 days after patch release
- **High severity**: Disclosed within 180 days after patch release
- **Medium severity**: Disclosed within 365 days after patch release
- **Low severity**: At the discretion of the maintainers

#### Public Disclosure

We believe in responsible disclosure and will:
- Provide advance notice to affected users before public disclosure
- Work with reporters to ensure accurate information is disclosed
- Credit researchers who follow our reporting process (if desired)
- Provide patches or workarounds before public disclosure

#### Coordinated Disclosure

For high-impact vulnerabilities affecting multiple stakeholders, we may implement a coordinated disclosure process. This allows:
- Time for users to apply patches
- Coordination with security researchers
- Minimized risk to end users

## Security Best Practices for Users

1. **Keep your device updated**: Regular updates include security patches
2. **Use strong authentication**: Enable all available security features
3. **Report suspicious activity**: Contact us if you notice anything unusual
4. **Data privacy**: Be cautious about inputting sensitive information
5. **Network security**: Use trusted networks when possible

## Safe Harbor for Researchers

This policy provides safe harbor for security researchers who:
- Report vulnerabilities through our designated channels
- Do not exploit vulnerabilities for personal gain
- Do not disclose vulnerabilities publicly without coordination
- Act in good faith to improve security
