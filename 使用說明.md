# Argus 使用說明

## 快速啟動（你只需要這些命令）

首次設定已全部完成（依賴、migrations、bootstrap superuser、frontend build、Google Client ID）。下次要用 Argus 時，**只要其中一種方式即可**：

### 模式 A：本機 runserver（最快）

```powershell
uv run python backend/manage.py runserver 127.0.0.1:8000
```

開瀏覽器：

| 用途 | 網址 |
|---|---|
| 前端 SPA（Google 登入） | http://127.0.0.1:8000/ |
| Django Admin | http://127.0.0.1:8000/admin/ |
| API（程式呼叫） | http://127.0.0.1:8000/api/... |

### 模式 B：Docker 完整堆疊（web + worker + redis + postgres + nginx）

```powershell
docker compose build
docker compose up
```

開瀏覽器：

| 用途 | 網址 |
|---|---|
| 前端 SPA（Google 登入） | http://localhost/ |
| Django Admin | http://localhost/admin/ |
| API | http://localhost/api/... |

### 兩種模式共用

- **一般使用者**：點「使用 Google 帳號登入」按鈕（首次自動建立帳號）。
- **系統管理員**：到上述 `/admin/` 網址，帳密 `1124` / `1124`。

### 修改了前端 (`frontend/src/*`) 之後

模式 A 需要重 build 才看得到：

```powershell
cd frontend ; npm.cmd run build ; cd ..
```

模式 B 重 `docker compose build` 就會順便 build frontend。

## 目前可用功能

- 一般使用者透過 Google 帳號登入或註冊（首次自動建立帳號）；系統管理員透過 Django Admin 以 username/password 登入。
- 使用者掃描配額（預設每月 20 次，可於 Django Admin 調整）。
- 建立授權式 ScanJob。
- 後端記錄授權證據：user、IP、timestamp、user-agent、授權網域。
- 明顯第三方或敏感網域需要二次確認。
- Passive / Active 模式檢查；Active 必須額外授權，並套用 RPS ≤ 2 速率限制。
- Playwright 同網域爬蟲：same-origin、最大深度、最大頁數、robots.txt、截圖、連結擷取、被阻擋頁面偵測（401/403/429）。
- 四維靜態掃描：SEO、AEO、GEO、被動資安，並含 GEO FAST 檢查（llms.txt、AI 爬蟲可存取性、JS 渲染依賴、語意化區塊、段落可引用性）。
- Findings、頁面資料、狀態查詢、截圖下載、Word 報告匯出。
- React 前台：Google 登入、建立掃描、任務列表、互動報告（findings 分類/嚴重度篩選、截圖高光對齊）、複製 AI handoff prompt、匯出 Word。
- Django Admin：掃描總覽儀表、Agent token 監控、授權同意書唯讀檢視。

## 第一次啟動

1. 複製環境設定：

```powershell
Copy-Item .env.example .env
```

2. 編輯 `.env`：

- `DJANGO_SECRET_KEY` 換成本機開發用 secret。
- `JWT_SECRET_KEY` 換成本機開發用 secret。
- 本機快速測試可保留：
  - `CELERY_TASK_ALWAYS_EAGER=true`
  - `ARGUS_AUTO_QUEUE_SCANS=true`

3. 安裝 Python 依賴：

```powershell
uv sync
```

4. 安裝 Playwright Chromium：

```powershell
$env:PLAYWRIGHT_BROWSERS_PATH = ".ms-playwright"; uv run playwright install chromium
```

注意：`PLAYWRIGHT_BROWSERS_PATH=.ms-playwright` 會把 Chromium 安裝在專案內，避免污染使用者層級快取。

5. 建立資料庫：

```powershell
uv run python backend/manage.py migrate
```

6. 安裝前端依賴：

```powershell
cd frontend
npm.cmd install
cd ..
```

## 啟動開發環境（進階：前端 hot-reload）

如果你正在改前端、想要 Vite hot-reload（不必每次 build）：

後端：

```powershell
uv run python backend/manage.py runserver 127.0.0.1:8000
```

前端 dev server：

```powershell
cd frontend
npm.cmd run dev
```

此時打開 `http://127.0.0.1:5173/`（Vite dev server）；frontend dev server 會 proxy API 請求到 backend 8000。一般使用情境用「快速啟動」模式 A 即可。

## Google OAuth 設定（首次部署必做）

一般使用者透過 Google 登入，需先取得 Google Cloud OAuth Client ID：

1. 進入 [Google Cloud Console](https://console.cloud.google.com/) 並選擇或建立一個專案。
2. 至「APIs & Services > OAuth consent screen」，設定使用者類型（External）與應用程式名稱。
3. 至「APIs & Services > Credentials > Create credentials > OAuth client ID」。
4. Application type 選 **Web application**。
5. Authorized JavaScript origins 加入：
   - `http://localhost:5173`（本機 Vite dev）
   - `http://localhost`（Docker compose 透過 nginx）
   - `http://127.0.0.1:5173`（備用）
6. 建立後取得 Client ID（形如 `xxx.apps.googleusercontent.com`），填入專案根目錄 `.env`：

```text
GOOGLE_OAUTH_CLIENT_ID=xxx.apps.googleusercontent.com
```

7. 重新啟動 backend 與 frontend；登入頁就會出現 Google 登入按鈕。

未設定 `GOOGLE_OAUTH_CLIENT_ID` 時，前端會顯示「尚未設定」訊息，後端 `/api/auth/google/` 會回 HTTP 503。

## 使用流程

1. 開啟前端網址（模式 A：http://127.0.0.1:8000/ 或模式 B：http://localhost/）。
2. 點選「使用 Google 帳號登入」（首次會自動建立帳號）。
3. 輸入你擁有或已取得書面授權的網址。
4. 勾選「我擁有此網站或已獲得書面授權測試」。
5. 若是第三方或敏感網域，必須再次確認授權。
6. 建立掃描。
7. 等待狀態更新為 `completed`。
8. 點選掃描任務查看 findings。
9. 點選 finding 可查看描述、修補方向，並複製 AI handoff prompt。
10. 點選「匯出 Word」下載報告。

```powershell

```

## 注意事項

- 不要把 `.env`、`GoogleCloud_ApiKey.json`、API Key、Token、私鑰提交到版本控制。
- 只掃描你擁有或取得書面授權的網站。
- 主動式資安測試只在額外授權後使用。
- 本系統只提供問題描述、證據、修補方向與 AI handoff prompt，不輸出修復後程式碼。

## Docker 本機部署（可選）

完整端到端容器化啟動（web/worker/redis/db/nginx），所有依賴都在 image 與 volume 內，不污染主機：

```powershell
docker compose up --build
```

啟動後：

- 前端（含 React Router）：`http://localhost/`
- Django Admin：`http://localhost/admin/`
- API：`http://localhost/api/`

說明：

- 本 compose 為本機開發配置；`DJANGO_DEBUG=true` 讓 runserver 自動服務 Admin 靜態檔。
- 正式部署需改用 gunicorn + `collectstatic` 並設 `DJANGO_DEBUG=false`。
- 容器內 Playwright Chromium 位於 `/ms-playwright`（與主機 `.ms-playwright` 各自獨立）。
- 資料庫資料持久化於 docker volume `postgres_data`，截圖與報告於 `media_data`。

## Cache replay 重跑掃描

修改 scanner 邏輯後，可不向目標站重發請求，直接用已儲存的 Page 內容重跑：

```powershell
uv run python backend/manage.py rerun_scan <scan_id>
```

加 `--keep-findings` 保留原 findings 僅追加新項。被阻擋（401/403/429）的頁面與站台層級 GEO FAST 訊號（llms.txt、AI 爬蟲）不參與 replay。
